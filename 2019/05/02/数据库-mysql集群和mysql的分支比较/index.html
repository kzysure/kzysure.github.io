数据库/mysql集群和mysql的分支比较

5月 2, 2019

集群

主要分为Replication和PXC两个方式，
区别：Replicaiton 性能好 分为主从，读写有延迟，适合社交等数据价值低 一致性要求低的系统 如贴吧等社交平台
PXC：高一致性，性能稍差，所有节点都可以读写，一致性高，适合数据价值高的系统 如金融系统/电商系统

分支

mysql主要分为三个版本
mysql官方：oracle官方，并发性能差，有闭源的风险，兼容性好
mariadb：由mysql之父开发的，完全开源，并发性能好于mysql，兼容性差（如不支持json格式）
perconna server：perconn 开发，性能最好，完全开源，并发性能最好，兼容性最佳（所有版本基于对应的mysql版本开发，如percona server 5.7基于mysql 5.7）阿里开发自有mysql组件时也参考了percona大量理念
